## Context

Search over a document collection usually returns links, not answers — and a raw
LLM will confidently invent facts. I built a retrieval-augmented generation (RAG)
assistant that answers questions in natural language while **citing the source
passages** it used, so every answer is checkable.

## Approach

RAG is a retrieval problem wearing a generation costume, so I spent most of the
effort on retrieval quality.

- Chunked documents with overlap and embedded them into a FAISS index for fast
  nearest-neighbor search.
- On a query, retrieve the top-k passages, then prompt the LLM to answer **only**
  from that context and to quote the passages it relied on.
- Served the whole thing behind a small FastAPI endpoint so it's callable from a
  simple web UI.

The key evaluation insight: I measured retrieval separately from generation. If
the right passage never makes it into the top-k, no amount of prompt tuning will
save the answer — so retrieval recall was the metric I optimized first.

## Result

Retrieval reaches **0.93 top-k recall** on a hand-labeled question set, and the
cited-sources design makes wrong answers easy to spot and correct — the assistant
shows its work instead of asking to be trusted.

## Reflection

Next I'd add a reranking step over the retrieved passages and an automated
"faithfulness" check that flags answers whose claims aren't supported by the
cited text — moving evaluation from manual spot-checks toward something
continuous.
